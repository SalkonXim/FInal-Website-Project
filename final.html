The CIA Framework

Data is a valuable asset. The primary goal of cybersecurity is to maintain three critical properties — collectively known as the CIA Triad — for all data, all the time.

Confidentiality

Data is not revealed

Protection against unauthorized access. Only those with proper permission should ever be able to read or view the data.

Think of it as a locked diary — only the owner (or those granted access) can read it.

REAL-WORLD BREACH

Edward Snowden's 2013 leak of classified NSA government data is a landmark example of a Confidentiality breach.

Integrity

Data is intact - not modified or corrupted

Protection against unauthorized modification. Data must remain accurate and trustworthy from the moment it is created until it is accessed.

If someone changes your grade in the grade book without permission, that's an Integrity breach.

REAL-WORLD BREACH

The Stuxnet worm secretly altered code on Iranian centrifuge machines, causing them to spin too fast and physically break — a classic Integrity attack.

Availability

Data is accessible to allowed users

Protection against denial of service. Authorized users must be able to access data and systems whenever they legitimately need them.

A website that is always down, or an account that stays locked, fails the Availability test.

REAL-WORLD BREACH

The Mirai Botnet used 145,000+ infected cameras and devices to flood Brian Krebs' website with traffic, crashing it for 4 days — a textbook DDoS / Availability breach.
Security Formula: Protection = Prevention + (Detection + Response)

Three States of Data

Data exists in different forms and each state requires specific protections.

Data at Rest

Data stored on hard drives, databases, USB devices, or cloud storage.

  • Protected with encryption
  • Requires strong access control
  • Examples: Files, backups, databases

Data in Transit

Data moving across networks or the internet.

  • Protected with HTTPS and VPNs
  • Can be intercepted without encryption
  • Examples: Emails, web traffic

Data in Use

Data actively being processed in memory or applications.

  • Protected through permissions
  • Vulnerable to malware attacks
  • Examples: Open documents, active sessions

Authentication & AAA

AAA stands for Authentication, Authorization, and Accounting.

Authentication

Verifies identity. Proves you are who you claim to be.

Authorization

Determines what resources and systems you can access.

Accounting

Tracks user actions and logs activity for auditing and security.

Password Security

Passwords remain one of the most common authentication methods.

Strong Passwords

  • Use 12+ characters
  • Mix uppercase, lowercase, numbers & symbols
  • Avoid dictionary words

Password Managers

Password managers securely store and generate complex passwords.

Common Attacks

  • Brute Force
  • Dictionary Attacks
  • Credential Stuffing
  • Phishing

Hashing

Hashing converts data into a fixed-length value used for verification and integrity.

What is Hashing?

A one-way mathematical function that transforms data into a unique hash value.

Popular Algorithms

  • SHA-256
  • SHA-3
  • bcrypt
  • Argon2

Password Storage

Passwords should never be stored in plain text — only hashed and salted.

Authentication Methods

Different authentication methods provide varying levels of security.

Something You Know

Passwords, PINs, passphrases, and security questions.

Something You Have

Security tokens, smartphones, smart cards, or OTP devices.

Something You Are

Biometrics such as fingerprints, face recognition, or retina scans.
